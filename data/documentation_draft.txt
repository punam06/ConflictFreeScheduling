Project Overview
A conflict-free class routine generator using advanced algorithms to optimize
scheduling for 4 batches (8 sections) with limited classroom resources.
1. Project Plan & Draft
1.1 Problem Analysis
• Constraints:
o 4 batches × 2 sections = 8 sections total
o 6-7 available classrooms
o Time slots: 8:30 AM - 3:30 PM
o Lunch break: 1:30-2:00 PM
o Class breaks: 15 minutes (optimize to 10-5 minutes)
o Theory classes: 3 credit hours, 2 sessions/week, 1 hour each
o Lab classes: 1.5 credit (3 hours weekly) or 0.75 credit (3 hours bi-weekly)
1.2 Core Components
1. Input Management System
2. Graph-based Conflict Detection
3. Graph Coloring Algorithm
4. Backtracking for Optimization
5. Database Integration (SQL)
6. PDF Generation
1.3 Data Structures Used
• Adjacency List for conflict graphs
• Hash Maps for quick lookup
• Priority Queues for scheduling optimization
• Trees for backtracking solutions
2. Algorithm Design
2.1 Graph Coloring Algorithm
Input: Courses, Time slots, Rooms
Output: Conflict-free schedule
1. Create conflict graph where:
- Nodes = Course sessions
- Edges = Conflicts (same teacher, same batch, room conflicts)
2. Apply Graph Coloring:
- Colors represent time slots
- Adjacent nodes cannot have same color
3. Optimize using Welsh-Powell algorithm
2.2 Backtracking Algorithm
BacktrackSchedule(courseIndex, assignment):
if courseIndex == totalCourses:
return true // Solution found
for each timeSlot in availableSlots:
if isValidAssignment(courseIndex, timeSlot):
assignment[courseIndex] = timeSlot
if BacktrackSchedule(courseIndex + 1, assignment):
return true
assignment[courseIndex] = null // backtrack
return false

4.2 SQL Schema
CREATE DATABASE RoutineGenerator;
CREATE TABLE batches (
batch_id INT PRIMARY KEY,
batch_name VARCHAR(20),
year_level INT
);
CREATE TABLE courses (
course_id INT PRIMARY KEY AUTO_INCREMENT,
course_code VARCHAR(10) NOT NULL,
credit_hours DECIMAL(3,2),
class_type ENUM('theory', 'lab'),
sessions_per_week INT,
batch_id INT,
FOREIGN KEY (batch_id) REFERENCES batches(batch_id)
);
CREATE TABLE teachers (
teacher_id INT PRIMARY KEY AUTO_INCREMENT,
teacher_code VARCHAR(20) UNIQUE,
availability_start TIME,
availability_end TIME,
external_faculty BOOLEAN DEFAULT FALSE
);
CREATE TABLE classrooms (
room_id INT PRIMARY KEY AUTO_INCREMENT,
room_code VARCHAR(20) UNIQUE,
capacity INT,
room_type ENUM('theory', 'lab', 'both'),
building VARCHAR(50)
);
CREATE TABLE time_slots (
slot_id INT PRIMARY KEY AUTO_INCREMENT,
start_time TIME,
end_time TIME,
day_of_week ENUM('SUN', 'MON', 'TUE', 'WED', 'THU')
);
CREATE TABLE schedule (
schedule_id INT PRIMARY KEY AUTO_INCREMENT,
course_id INT,
teacher_id INT,
room_id INT,
slot_id INT,
section ENUM('A', 'B'),
FOREIGN KEY (course_id) REFERENCES courses(course_id),
FOREIGN KEY (teacher_id) REFERENCES teachers(teacher_id),
FOREIGN KEY (room_id) REFERENCES classrooms(room_id),
FOREIGN KEY (slot_id) REFERENCES time_slots(slot_id)
);
5. Core Algorithms Implementation
5.1 Graph Coloring (Welsh-Powell)
1. Sort vertices by degree (descending)
2. Assign colors greedily
3. Optimize for minimum colors used
5.2 Conflict Detection
• Teacher conflicts: Same teacher, same time
• Room conflicts: Same room, same time
• Batch conflicts: Same batch, same time
• External faculty constraints: Limited availability
5.3 Optimization Techniques
• Heuristic 1: Schedule high-degree courses first
• Heuristic 2: Prioritize external faculty constraints
• Heuristic 3: Minimize gap times between classes
• Heuristic 4: Batch similar courses together

7. Performance Analysis
7.1 Time Complexity
• Graph Construction: O(n²) where n = number of courses
• Graph Coloring: O(n² log n)
• Backtracking: O(k^n) worst case, where k = time slots
• Overall: O(n² log n) average case with heuristics
7.2 Space Complexity
• Graph Storage: O(n²)
• Schedule Storage: O(n)
• Overall: O(n²)
